# PLAN: Feedback-Loop & Issue-Tracking

**Ziel:**
Automatisierte und manuelle Rückmeldung vom User erfassen, auswerten und in Verbesserungen oder Prioritäten übersetzen.

**Meilensteine:**
- M1: Periodischer Self-Feedback-Prompt (Telegram, Canvas, CLI)
- M2: Automatisches Sammeln/Aggregieren von Fehlern, Timeouts, Incidents aus Logs
- M3: Review/Ranking der wichtigsten Feedbacks, Priorisierung ins ToDo/Backlog
- M4: Success-Report und Lessons Learned ins Keeper/Digest

**Success-Kriterien:**
- Kein wertvolles Feedback geht verloren
- Verbesserungen entstehen gezielt aus User-Feedback
