# PLAN: Integrity & Recovery

**Ziel:**
Dauerhafter Systemschutz und Wiederherstellbarkeit – Agent erkennt Integritätsprobleme/frühe Fehler und validiert Notfallpläne, Backups, Policies.

**Meilensteine:**
- M1: Routine-Selftest für Skills, Cronjobs, Netzwerk und Kernservices (Agent erledigt das autonom)
- M2: Simulation Probe-Backup/Restore; Validierung und Recovery-Option vorbereiten
- M3: Anomaly-Detection für Service-Stati und Policies

**Success-Kriterien:**
- Integritätsverlust wird früh erkannt, Agent schlägt von sich aus Maßnahmen vor
- „Disaster Recovery“ ist jederzeit getestet und nachvollziehbar dokumentiert
