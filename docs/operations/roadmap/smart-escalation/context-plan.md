# PLAN: Context-Aware Policies & Adaptive Actions

**Ziel:**
Agent passt Policies und Aktionen dynamisch an Zustand, Kontext (Nacht/Tag, Anwesenheit, Meetings etc.) und aktuelle Needs an.

**Meilensteine:**
- M1: Policy-Template-Engine mit kontextsensitiven Variablen/Regeln
- M2: Detection-API für Zeitslot, Userpräsenz, Energie/Last, wichtige externe Trigger
- M3: Härtung durch laufende Policy- und Schwellenanpassung
- M4: „Quiet Mode“/Notification-Tuning abhängig von Umwelt/Events

**Success-Kriterien:**
- Policies passen sich ohne manuellen Eingriff, kontextbewusst immer optimal an
