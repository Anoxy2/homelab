# PLAN: Smarte Eskalation

**Ziel:**
Alarmmuster und Benachrichtigung so steuern, dass nur kritisches sicher (und passend gestaffelt) beim User landet.

**Meilensteine:**
- M1: Eskalations-Schwellen und Regeln im Keeper definieren (z. B. Stille in der Nacht, Priority-Klassen, User-Präsenz)
- M2: Escalation-Skill bauen (Entscheidungslogik, Push/Log/Autoaktion)
- M3: Schwellen dynamisch anpassen und mit User-Feedback harmonisieren
- M4: Meldeweg und Verlauf ins Keeper-Protokoll

**Success-Kriterien:**
- Kaum unnötige Notifications, keine False Negatives
- Alarmierungsverlauf jederzeit nachvollziehbar
